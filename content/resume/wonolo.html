I joined Wonolo as an inaugural member of the newly-formed Build and Release Tools team. At the intersection of build and release engineering, platform engineering, and DevOps, our team’s purview was maintaining and improving the tooling and processes that assisted developers in releasing software more rapidly and more safely.

  • Responsible for stewardship of our CI/CD solution and assisting other developers in utilizing it successfully.
  • Frequent liaison between CloudOps (i.e. the infrastructure team) and developer teams.
  • Drove and participated in many inter-team discussions to drive best practices, process improvements, and developer experience improvements.

Highlights

  • Designed and implemented a (Ruby) tool for developers to formally document code ownership. The tool parsed multiple sources of truth into ownership trees consisting of files and code objects, reconciled and grafted them into a singular tree, and generated a canonical CODEOWNERS file for GitHub to resolve code ownership.
  • Designed and implemented a (Node.js) service that generated recommendations for pull request reviewers to invite based on data from a GitHub repository’s CODEOWNERS file and other custom criteria. The service integrated with any configured GitHub repositories and published recommendation reports as comments on open pull requests.
  • Maintained and improved an internal (Python) deployment queue service — integrated between GitHub, Slack, and our CI/CD platform — to assist developers in collaboratively managing builds/deployments. The service handled locking of exclusive deployment resources as well as provided monitoring feedback to inform developers of pipeline progress. Developers interacted with the service through a Slackbot, allowing them to manage deployments with a minimum of additional panel-watching.
  • Wrote many unit and integration tests for all of the above projects.
