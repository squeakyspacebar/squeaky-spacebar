I joined the Dallas Morning News as an inaugural member of their new in-house developer team, where I primarily worked on designing and building a graph-based knowledge system — using Amazon Neptune, Elasticsearch, and GraphQL — for powering features across the DMN website.

Highlights

  • Built the core software (Node.js) for an internal knowledge graph system backed by Amazon Neptune and Elasticsearch.
  • Wrote an object-graph mapping (OGM) library that generated Gremlin queries (graph traversal language used by Amazon Neptune) for Node.js applications.
  • Wrote a preprocessor tool (Node.js) that dynamically generated the entire definition for the knowledge graph system's GraphQL API and stood up the API server. The tool utilized GraphQL as a DDL (data definition language), allowing less backend-oriented developers to submit updates to the API's data model by simply modifying GraphQL schema definition files, and reducing the load on me in managing API updates.
  • Helped develop and maintain knowledge graph systems such as the administrative interface and AWS infrastructure.
  • Managed implementation, ETL, data cleaning, and data processing for a number of content features that utilized the knowledge graph including the North Texas Giving Guide, Top 100 Places to Work, and School Guide.
  • Assisted in migrating dallasnews.com from a custom CMS onto the Washington Post's Arc Fusion (React-based) platform for a site relaunch.
  • Assisted in developing React components for the DMN website, often centered around event search functionality that utilized the knowledge graph system.
