At the Dallas Morning News, I primarily worked on building LocalGraf, a graph-based knowledge system backed by AWS Neptune and GraphQL intended to power features across the DMN website. I designed and wrote much of the fundamental software for LocalGraf.

  • Wrote an object graph mapper (OGM) library called Mogwai that generated the necessary Gremlin queries to talk to AWS Neptune.
  • Wrote a library called Pegasis that parsed an object schema as a set of GraphQL type definitions, dynamically generated out the entire expanded GraphQL query/mutation schema that actually defined the operational API, attached the proper Mogwai handler functions to the generated endpoints, and stood up the API server. It basically dynamically generated the entire API. Generating from a set of GraphQL schema definition files allowed other, more front end oriented developers easily submit their own schema modifications via PR review with little necessary training.
  • Managed implementation, ETL, and data quality for a number of additional features that utilized LocalGraf including the North Texas Giving Guide, Top 100 Places to Work, and School Guide.
  • Assisted in creating/maintaining various front end React components for the main DMN website, mostly centered around Things to Do event search functionality since it tied heavily into LocalGraf.
  • Helped address the challenges arising from the rabbit-hole that is searching scheduled events by date.
  • Discovered just what kind of developer nightmare the Washington Post's Arc Publishing platform is.
